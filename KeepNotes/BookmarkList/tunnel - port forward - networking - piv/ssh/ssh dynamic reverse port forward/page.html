So you have access to a machine, but it's not ssh access and ssh isn't open from the outside. Nifty trick:

Step1: create a dynamic port forward to localhost
Step2: reverse forward the newly created dynamic port to your attacking machine

This does only work if the target machine  has an ssh server and is running



Newer versions of SSH:
OpenSSH 7.6 introduced reverse dynamic proxy as a native option. It is implemented entirely in the client, so the server does not need to be updated.
ssh -R 1080 server
